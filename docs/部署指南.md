# 部署指南

> 本文档说明如何从零部署 Mimir，包括 vLLM in-tree patch 的接入与 Benchmark 复现。

## 1. 环境要求

### 1.1 操作系统（满足赛题要求）

- openEuler（推荐）、openKylin、OpenHarmony
- 其他 Linux 发行版（Ubuntu 22.04+、CentOS 等）

### 1.2 软件依赖

| 依赖 | 版本 | 说明 |
| --- | --- | --- |
| Python | 3.11（conda 环境 `mimir`） | 推荐 3.11 |
| conda | Miniconda/Anaconda | 环境 |
| Git | ≥ 2.25 | SSH clone（HTTPS 可能被屏蔽） |
| CUDA 驱动 | ≥ 12.8 | GPU 推理；torch 2.8.0+cu128 |
| pip | ≥ 24 | |

### 1.3 硬件

- **最低**：8GB 主存 + CPU（仅跑逻辑测试 + 仿真）
- **推荐**：≥ 16GB 显存 GPU（RTX 3090/4090 / A10），用于 vLLM + Benchmark
- **异构**：昇腾（DTK/CANN）/ 寒武纪等（`mimir.hardware` 有探测钩子）

### 1.4 模型（本地已有或自行下载）

| 模型 | 路径 | 用途 |
| --- | --- | --- |
| Qwen3-1.7B | `/data/models/Qwen3-1.7B` | 泛化验证（小） |
| Qwen3-4B-Instruct-2507 | `/data/models/Qwen3-4B-Instruct-2507` | **主力** |
| Qwen3-8B | `/data/models/Qwen3-8B` | 泛化验证（大，需更大显存） |

## 2. 从零部署

### 2.1 克隆仓库

```bash
git clone git@github.com:TobyMint/Mimir.git   # SSH（HTTPS 被屏蔽）
cd Mimir
```

### 2.2 创建 conda 环境

```bash
conda create -n mimir python=3.11 -y
conda activate mimir
pip install torch==2.6.0 torchvision==0.21.0 torchaudio==2.6.0 \
    --index-url https://download.pytorch.org/whl/cu126
pip install setuptools_scm
pip install -e ".[dev]"
```

> torch 2.8.0+cu128 也可（`pip install torch==2.8.0 --index-url .../cu128`），与 vLLM 0.10.2 的 ABI 匹配。

### 2.3 接入 vLLM（普通目录 + 预编译二进制）

vLLM v0.10.2 源码已在 `third_party/vllm/`（普通 tracked 目录，非 submodule）。
纯 Python patch 直接改 `third_party/vllm/vllm/v1/...`，不重编 `_C`。

预编译二进制（`.abi3.so` + `vllm_flash_attn`）不入库，需一次性重建：

```bash
bash scripts/setup_vllm_binaries.sh   # 下载 vllm==0.10.2 wheel，提取 .so，symlink 进 third_party/vllm/
```

### 2.4 激活环境（每次会话）

```bash
source scripts/activate_env.sh
```

该脚本幂等地做四件事：
1. `conda activate mimir`
2. 写 `.pth` → `import vllm` 解析到 `third_party/vllm`
3. 写 `vllm-0.10.2.dist-info` → `importlib.metadata.version("vllm")` 返回 0.10.2（vLLM 平台检测依赖）
4. `LD_LIBRARY_PATH += torch/lib` + `VLLM_USE_V1=1` `VLLM_ENABLE_V1_MULTIPROCESSING=0`（v1 单进程）

### 2.5 验证

```bash
make reproduce    # CPU 模式（--quick）：110 测试 + 仿真 ~2 分钟
```

## 3. Benchmark 复现

### 3.1 Agent-Loop Benchmark（赛题核心评测）

```bash
source scripts/activate_env.sh
python scripts/run_agent_benchmark.py --tool-kb 5 --max-tokens 96
```

跑 3 个真实 agent 任务（research / compare / estimate），每个任务是一个真正的 agent loop：
LLM 生成 → 解析 tool_call → 执行 mock 工具 → 结果喂回 → LLM 再生成。

A/B 对照：native（fcfs，工具结果全量进 KV）vs Mimir（mimir 策略 + 工具外置 + 自动回收）。

输出：`benchmark_results/agent_benchmark_<model>.json` + `_curves.png`（英文标签）

### 3.2 单特性 A/B

| 命令 | 验证 |
| --- | --- |
| `python scripts/run_phase2_context.py` | 上下文压缩 TTFT -93% |
| `python scripts/run_phase3_offload.py` | 工具外置 new_prefill -83% |
| `python scripts/run_phase4_branch.py` | 分支 CoW 真实 APC 复用 |
| `python scripts/run_phase_blockclass.py` | **创新**：block-class 类别感知淘汰（evict 只淘汰 reasoning）|
| `python scripts/run_recall_metric.py` | 诚实指标：tool-call probe 召回率 |
| `python scripts/run_trace_benchmark.py` | DeepSeek 真实轨迹 A/B（native 崩 vs Mimir 跑完）|
| `python scripts/run_llm_judge.py` | DeepSeek-judge 压缩保真 A/B |

### 3.3 全量复现

```bash
bash scripts/reproduce.sh           # 含 GPU 验证
bash scripts/reproduce.sh --quick   # 仅 CPU
```

## 4. 在 Agent 工作流中使用

```python
from mimir.manager import MemoryManager
from mimir.engine_vllm_v1 import VLLMEngineV1
from mimir.engine_vllm import EngineConfig

eng = VLLMEngineV1(EngineConfig(
    model="/data/models/Qwen3-4B-Instruct-2507",
    dtype="bfloat16",
    gpu_memory_utilization=0.55,
    enable_prefix_caching=True,
    extra={"scheduling_policy": "mimir"},
), device=0)

mm = MemoryManager(features=["context_compress", "tool_offload", "prefix_cache", "tiered", "lifecycle"])

# 统一入口：外部压缩/外置 + 引擎自动回收
result = mm.run_turn_with_engine(eng, case, task_id="t1", max_tokens=128)
```

## 5. 常见问题

| 问题 | 解决 |
| --- | --- |
| `import vllm` 报错 | 先 `source scripts/activate_env.sh`（写 .pth + dist-info） |
| `Device string must not be empty` | dist-info 未写；重跑 `activate_env.sh` |
| `libtorch.so not found` | `LD_LIBRARY_PATH` 未含 torch/lib；`activate_env.sh` 会自动设 |
| GitHub HTTPS 被屏蔽 | 使用 SSH（`git@github.com:`） |
| 显存不足 | 降低 `gpu_memory_utilization`，或用 Qwen3-1.7B |
| fp8 报错 | RTX 3090 不支持 fp8，Phase F 会自动降级 bf16 |

## 6. 复现性保证

- 优化前后使用**完全相同的硬件与模型**（赛题要求）
- Benchmark 脚本固化于 `scripts/` + `benchmarks/`，含随机种子
- 环境与版本通过 `pyproject.toml` + conda 环境 `mimir` 锁定
- vLLM patch 清单可追溯：[`docs/VLLM_PATCH_INVENTORY.md`](VLLM_PATCH_INVENTORY.md)
