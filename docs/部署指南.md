# 部署指南

> 本文档说明如何在不同环境（openEuler / openKylin / 通用 Linux）下从零部署 Mimir。
> 状态：**初稿**，随实现推进补充具体命令与版本。

## 1. 环境要求

### 1.1 操作系统（满足赛题要求）

- ✅ openEuler（推荐，赛题首选之一）
- ✅ openKylin
- ✅ OpenHarmony / 其他 Linux 发行版（Ubuntu 22.04+、CentOS 等）

### 1.2 软件依赖

| 依赖 | 版本 | 说明 |
| --- | --- | --- |
| Python | ≥ 3.9 | 推荐 3.10 / 3.11 |
| GCC / G++ | ≥ 9 | 编译 C++ 扩展 / llama.cpp |
| CMake | ≥ 3.18 | llama.cpp 构建 |
| Git | ≥ 2.20 | LFS 拉取大文件 |
| CUDA Toolkit | 11.8 / 12.x | 可选，GPU 推理 |

### 1.3 硬件

- **最低**：8GB 主存 + CPU（仅 llama.cpp + offload 验证逻辑）
- **推荐**：≥ 16GB 显存 GPU（如 RTX 3090/4090 / A10），用于 vLLM + Benchmark
- **异构**：昇腾（DTK/CANN）/ 寒武纪等（详见 [`技术方案.md`](技术方案.md) §3.6）

## 2. 安装步骤

### 2.1 克隆仓库

```bash
# SSH（推荐，环境内 HTTPS 可能被屏蔽）
git clone git@github.com:TobyMint/Mimir.git
cd Mimir
```

### 2.2 创建虚拟环境

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
```

### 2.3 安装 Mimir

```bash
# 仅核心
pip install -e .

# 含开发工具链
pip install -e ".[dev]"

# 含推理后端（按需）
pip install -e ".[vllm]"      # vLLM 后端
pip install -e ".[llamacpp]"  # llama.cpp 后端
```

### 2.4 准备模型

模型权重较大，不在仓库中托管，需自行下载（参考 [`赛题说明.md`](赛题说明.md) 参考资料）：

- **MiniCPM3-4B**：https://www.modelscope.cn/models/OpenBMB/MiniCPM3-4B
- **Qwen 系列**：https://www.modelscope.cn/models/qwen

```bash
# 下载示例（ModelScope）
pip install modelscope
python -c "from modelscope import snapshot_download; snapshot_download('OpenBMB/MiniCPM3-4B')"
```

将模型路径配置到 `benchmarks/config.yaml`（见下）。

## 3. 运行

### 3.1 单元测试

```bash
make test          # 全部
make test-fast     # 跳过 slow / gpu
```

### 3.2 Benchmark（赛题复现）

```bash
make benchmark     # 等价 python -m benchmarks.run
```

结果输出至 `benchmark_results/`，详见 [`测试报告.md`](测试报告.md)。

### 3.3 在 Agent 工作流中使用

```python
from mimir import MemoryManager

mm = MemoryManager(backend="vllm", features=["prefix_cache", "branch_cow"])
# ...见 docs/系统设计.md §3.3
```

## 4. 常见问题

| 问题 | 解决 |
| --- | --- |
| GitHub HTTPS 被屏蔽 | 使用 SSH（`git@github.com:` 前缀） |
| vLLM 安装失败 | 按官方文档安装匹配 CUDA 版本，或改用 llama.cpp 后端 |
| 显存不足 | 启用 `tiered` 分层存储，或减小 batch / 上下文 |
| 异构后端报错 | 确认 DTK/CANN 环境，或降级到 CPU |

## 5. 复现性保证

- 优化前后使用**完全相同的硬件与模型**（见 [`赛题说明.md`](赛题说明.md) 赛题要求）
- Benchmark 脚本固化于 [`../benchmarks/`](../benchmarks/)，含随机种子
- 环境与版本通过 `requirements.txt` 与 `pyproject.toml` 锁定
